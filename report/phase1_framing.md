# Phase 1 — Framing and Scope

## Main Research Question

**What does the "AI-augmented software development lifecycle / AI-gile" look like with coding agents being the main producer of code? How do software teams adapt processes towards shipping production-level software?**

## Domain

**Future of work** — specifically, software development.

## Sub-Questions

1. **Process adaptation**: How do software teams change their workflows when AI coding assistants become primary code producers?
2. **Quality and trust**: What quality assurance, testing, and validation strategies emerge for AI-generated code?
3. **Developer roles**: How do developer roles (junior, senior, architect) shift in AI-augmented teams?
4. **Tool integration**: What tools, practices, and guardrails support reliable AI-assisted development?
5. **Barriers and adoption**: What technical, organizational, and ethical barriers affect adoption of AI coding tools?

## Inclusions

- Peer-reviewed papers (ACM, journals) and preprints (arXiv) on AI-augmented software development
- Empirical studies on developer productivity, code quality, and workflow changes
- Technical reports on coding agents, LLMs in SE, and model-driven development with AI
- Papers discussing testing, code review, and quality metrics for AI-generated code
- Sources from 2015 onward (foundational + recent)

## Exclusions

- Papers focused solely on general-purpose LLMs without software-engineering context
- Hardware or infrastructure papers (e.g., GPU scaling) unless directly tied to SE tooling
- Non-English sources
- Purely theoretical AI/ML papers without SE application

## Scope Discipline

- **Corpus size**: 15–30 sources minimum
- **Peer-reviewed minimum**: At least 8 peer-reviewed papers or technical reports
- **Geographic scope**: No restriction; international corpus
- **Temporal scope**: Emphasis on 2020+ for adoption/empirical work; foundational papers from earlier years allowed
